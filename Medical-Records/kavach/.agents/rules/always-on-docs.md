# Always-On Docs Rule

**Type:** always-on
**Scope:** all files

Before making any changes to this project, you MUST:

1. Read `docs/project-brief.md` to understand the product definition and v1 scope.
2. Read `docs/architecture.md` to understand the system design and module boundaries.
3. Read `docs/decisions.md` to understand what has already been decided.
4. Read `docs/tasks.md` to understand what is currently in scope.
5. Read `docs/progress.md` to understand the current project state and blockers.

After completing any meaningful work:

1. Update `docs/progress.md` with the new state.
2. Update `docs/tasks.md` to mark completed items and add next steps.
3. Update `docs/changelog.md` with a dated entry.
4. Update `docs/handoff.md` so the next session has full context.
5. If architecture or design changed, update `docs/architecture.md` and add an ADR to `docs/decisions.md`.

If docs and code disagree, resolve the contradiction immediately before continuing.
