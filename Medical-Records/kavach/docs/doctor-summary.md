# Doctor Summary — Kavach

> Last updated: 2026-05-16

---

## Purpose

The one-page doctor summary is the primary value delivery artifact of Kavach. It is shared by the caregiver with the doctor via WhatsApp before or during a consultation. It must be readable on a phone screen and printable on A4 paper in one page.

---

## Summary Sections

### Section 1: Patient Snapshot
| Field | Notes |
|---|---|
| Name | Full name as on records |
| Age | Calculated from date of birth |
| Blood group | From profile (editable by caregiver) |
| Weight (kg) | Most recent value from records |
| Known allergies | List from profile |

### Section 2: Active Conditions
| Field | Notes |
|---|---|
| Diagnosis name | From consultation records |
| Diagnosed date | Earliest record date for condition |

Sorted by recency. Show all active conditions (no hard limit, but compact display).

### Section 3: Current Medications
| Field | Notes |
|---|---|
| Drug name | Generic name preferred |
| Dosage | e.g., 500mg |
| Frequency | e.g., Twice daily |
| Prescribing doctor | From prescription record |

Only active medications (not discontinued). Flag if no recent prescription found.

### Section 4: Lab Trends
| Field | Notes |
|---|---|
| Primary markers | HbA1c, fasting glucose, eGFR, CBC, lipid panel — configurable |
| Condition-linked markers | Based on active conditions (provisional until HCP-validated) |
| Trend indicator | ↑ (above range), ↓ (below range), ✓ (within range) |
| Last value + date | Most recent test result |
| Previous value + date | Second most recent (for trend comparison) |

> **Important:** Lab marker sets are provisional until clinically validated by HCP interviews. Do not hardcode as final. Mark clearly in UI and code as `PROVISIONAL`.

#### Default marker set (provisional)
| Condition | Markers |
|---|---|
| Diabetes | HbA1c, Fasting Glucose, Post-Prandial Glucose, eGFR, Microalbumin |
| Hypertension | Creatinine, eGFR, Potassium, Sodium |
| Dyslipidaemia | Total Cholesterol, LDL, HDL, Triglycerides |
| Thyroid | TSH, T3, T4 |
| General | Haemoglobin, WBC, Platelets |

### Section 5: Recent Visits
Last 3 consultations:
| Field | Notes |
|---|---|
| Doctor name | From consultation record |
| Date | Consultation date |
| Diagnosis / chief complaint | From record |

### Section 6: Upcoming / Pending
| Field | Notes |
|---|---|
| Next review date | Manually entered by caregiver or extracted from consultation note |
| Pending tests | Tests ordered but not yet uploaded |

### Section 7: Portal Access
| Field | Notes |
|---|---|
| QR code | Encodes the time-limited portal link |
| Portal link | Text form of the link |
| Expiry notice | "Valid for 8 hours from generation" |

## PDF Layout Guidance

### Paper / screen target
- A4 portrait, print-safe (printable on any printer)
- Must also be readable on a 6-inch phone screen without zooming
- Single page maximum — no second page

### Visual hierarchy
1. Patient Snapshot — top header band (prominent)
2. Active Conditions + Current Medications — left column
3. Lab Trends — right column
4. Recent Visits — full width below columns
5. Upcoming/Pending — half width
6. Portal Access (QR + link) — bottom right corner

### Typography and design
- Font: Clean sans-serif (Inter or similar)
- Patient name: largest text element
- Section headers: bold, coloured (brand colour, accessible contrast)
- Values: clear, monospace for numbers
- Trend indicators: colour-coded (↑ red, ↓ blue, ✓ green) — also use symbol for print accessibility
- All text ≥ 10pt for print legibility
- Logo + "Generated by Kavach" footer

### Doctor usability expectations
- Doctor can scan the full summary in under 60 seconds
- No medical jargon abbreviations without expansion
- All reference ranges shown alongside values
- Trend direction obvious without requiring comparison

---

## Mobile Readability Requirements

- PDF viewed on WhatsApp mobile must be readable without pinch-zoom for key data
- Critical sections (patient snapshot, medications, lab trends) must be visible in first scroll
- Portal QR code must be scannable on a phone screen at full size

---

## Generation Trigger

1. Caregiver opens patient detail in PWA
2. Taps "Generate Summary"
3. System runs RAG pipeline → structured summary JSON
4. System renders PDF from template + summary JSON
5. PDF stored in S3
6. Pre-signed URL shown to caregiver for preview
7. Caregiver taps "Share via WhatsApp" — bot sends PDF to caregiver's WhatsApp
8. AccessToken created with 8-hour TTL
9. QR + link appended to PDF (re-rendered) and sent
