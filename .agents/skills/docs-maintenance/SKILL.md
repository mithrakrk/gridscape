# Skill: Documentation Maintenance

## Instructions
- Read all `docs/*.md` files relevant to the current task.
- Reconcile stale status: if a task is done in code but not in docs, fix the docs.
- Update `docs/progress.md`, `docs/tasks.md`, `docs/handoff.md`, `docs/changelog.md`, and `docs/decisions.md` when applicable.
- Keep entries dated (use YYYY-MM-DD format).
- Keep next steps explicit in `docs/handoff.md` and `docs/tasks.md`.
- **Never overwrite useful historical context**: If a file gets too large, summarize the top and append the older context below, or move it to an archive section. Do not simply delete history.
