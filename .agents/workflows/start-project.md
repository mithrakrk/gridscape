# Workflow: Start Project

## Purpose
To initialize a new project from a rough idea into a structured plan.

## Steps
1. **Gather Idea**: Read the user's initial project idea.
2. **Draft Brief**: Populate `docs/project-brief.md` with objectives, scope, and success metrics.
3. **Design Architecture**: Draft the initial technical approach and stack in `docs/architecture.md`.
4. **Create Tasks**: Break down the first phase of work into actionable items in `docs/tasks.md`.
5. **Review**: Present the brief, architecture, and tasks to the user for approval.
6. **Finalize**: Update `docs/progress.md` to indicate project start.
