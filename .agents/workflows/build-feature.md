# Workflow: Build Feature

## Purpose
To systematically implement a new feature from planning to documentation.

## Steps
1. **Read Context**: Consult `AGENTS.md`, `docs/project-brief.md`, `docs/architecture.md`, and `docs/tasks.md`.
2. **Plan**: Outline the technical implementation plan. Update `docs/tasks.md` with sub-tasks if necessary.
3. **Implement**: Write the code for the feature.
4. **Test**: Write and run tests to verify the implementation. Ensure readable test results.
5. **Update Docs**:
   - Mark task as done in `docs/tasks.md`.
   - Add the next recommended task to `docs/tasks.md`.
   - Update `docs/progress.md` and `docs/changelog.md`.
   - If architecture changed, update `docs/architecture.md` and `docs/decisions.md`.
   - Update `docs/handoff.md` with current state.
