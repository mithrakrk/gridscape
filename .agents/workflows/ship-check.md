# Workflow: Ship Check

## Purpose
To review the project state before a release or major milestone.

## Steps
1. **Review Readiness**: Check `docs/tasks.md` to ensure all critical tasks for the milestone are complete.
2. **Identify Blockers**: Note any open bugs or incomplete tests.
3. **Check Documentation Freshness**: Ensure `docs/changelog.md` accurately reflects the work done. Verify `docs/project-brief.md` and `docs/architecture.md` are up to date.
4. **Report**: Present a summary of readiness, blockers, and documentation status to the user.
